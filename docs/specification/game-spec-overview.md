# ゲーム仕様概要

## 目的
- Abyss Branch のゲーム仕様群を、参照順と責務で整理した入口文書を定義する。
- 仕様追加時に、どの文書へ何を記述すべきかの判断基準を固定する。

## 対象範囲
- ゲーム仕様文書の分類（game/system/data）。
- 各仕様文書の責務境界と参照順。
- 仕様変更時の更新ルール。
- 個別カード効果、個別敵挙動、個別数値の詳細定義は対象外とする。

## 本文
- 仕様文書は次の3分類で管理する。
  - `game`: プレイヤー体験に直結するルール（戦闘進行、報酬、成長、失敗条件）。
  - `system`: 実行基盤に関わる仕様（状態遷移、セーブ、乱数、同期要件）。
  - `data`: コンテンツ定義の構造（データスキーマ、ID規約、検証条件）。
- 参照順は `game -> system -> data` を基本とし、体験要件を先に確定する。
- 分類をまたぐ仕様は、主責務を1つに固定し、他分類には参照リンクのみ記述する。
- 既存仕様と競合する変更は、先に競合点を明示してから本文を改定する。

## 決定事項
- 新規仕様作成時は、最初に本書で対象分類を確定する。
- 仕様変更時は、影響する分類文書を同一作業で更新する。
- 実装都合で仕様を変更しない。仕様変更が必要な場合は、変更理由を文書に明記する。
- 仕様本文に暫定事項が残る場合は、`TODO` を明示して確定事項と区別する。
