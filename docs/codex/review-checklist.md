# 文書レビュー・チェックリスト

## 命名
- [ ] ファイル名は `kebab-case` になっている
- [ ] 拡張子は `.md` で統一されている

## 構成
- [ ] 必須見出し（目的/範囲/本文/決定事項または手順）がある
- [ ] 見出し順が規約と一致している
- [ ] 箇条書きの粒度が揃っている
- [ ] 入れ子構造を使う場合、見出し深さは `H4` 以内
- [ ] 文書内に `Update History` / `更新履歴` 見出しがない

## 内容
- [ ] 用語が文書内で統一されている
- [ ] 主要用語が `docs/policy/glossary.md` と一致している
- [ ] 曖昧な表現に注記または TODO がある
- [ ] 決定事項または手順が明確に書かれている

## 運用
- [ ] 変更が関連文書（README/Rules/Template）に反映されている
- [ ] 変更内容が `CHANGELOG.md` に反映されている
- [ ] lint ルールに違反していない
- [ ] コミットメッセージの型と書式が `docs/policy/commit-message.md` に一致している
- [ ] 既存文書の変更は差分提示と合意を経ている
