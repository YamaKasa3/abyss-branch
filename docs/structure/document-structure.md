# 文書構成

## 目的
`docs/` 配下の文書構造と責務を定義する。

## ルート構成
```text
docs/
├─ codex/
├─ specification/
├─ policy/
├─ workflow/
└─ structure/
```

## ディレクトリ責務
- `codex/`
  - Codex 運用ルール、テンプレート、レビュー基準を管理する。

- `specification/`
  - ゲーム仕様、システム仕様、データ仕様を管理する。

- `policy/`
  - 設計方針、データ方針、運用上の制約を管理する。

- `workflow/`
  - 開発・検証・リリースの作業フローを管理する。
  - 標準開発フローは `docs/workflow/development-flow.md` を正本とする。

- `structure/`
  - リポジトリ構造と文書構造の定義を管理する。

## 文書ルール
- 新規文書は責務に対応するディレクトリへ配置する。
- 文書名は `kebab-case` を使う。
- 順序が必要な文書のみ番号プレフィックスを付与する。
- 主要用語は英語で統一し、必要時のみ初出で日本語補足する。

## 更新ルール
- 構造変更時は `README.md` と本ファイルを同時更新する。
- 運用ルール変更時は `codex/rules.md` と整合性を取る。
- 変更履歴は文書内に持たず、`CHANGELOG.md` に集約する。
