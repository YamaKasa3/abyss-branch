# リポジトリ構成

## 目的
Abyss Branch リポジトリのトップレベル構成と、各ディレクトリの責務を定義する。

## トップレベル構成
```text
abyss-branch/
├─ CHANGELOG.md
├─ README.md
├─ godot/
├─ data/
├─ sim/
├─ tools/
└─ docs/
```

## ディレクトリ責務
- `README.md`
  - リポジトリ全体の概要と入口を示す。

- `CHANGELOG.md`
  - リリース単位の変更履歴（自分向け）を記録する。
  - プレイ体験や仕様に影響する変更を中心に記録する。

- `godot/`
  - ゲーム本体を実装する。
  - 実行ロジック、シーン、アセットを管理する。

- `data/`
  - カード・敵・報酬などのゲームデータを管理する。
  - バランス調整の基準となる正本データを保持する。

- `sim/`
  - Godot ヘッドレス実行の制御を行う。
  - シミュレーション実行と結果分析を行う。

- `tools/`
  - 開発補助処理を行う。
  - 検証、変換、生成などの非本体処理を管理する。

- `docs/`
  - 仕様・方針・運用ルールの正本ドキュメントを管理する。

- `docs/policy/commit-message.md`
  - コミットメッセージの型と書式のルールを管理する。

## ルール
- ゲームロジックは `godot/` を正とする。
- シミュレーション用途でも、ゲームロジックを別実装で重複させない。
- データ変更時は `data/` を起点に整合性を確認する。
- 変更履歴は文書内に持たず、`CHANGELOG.md` に集約する。
